We will be investigating age bias in ML models. This is relevant because ageism, or discrimination based on age, pervades many facets of American society. For example, in hiring in the tech industry, older applicants are typically rejected in favor of younger applicants [https://online.ucpress.edu/collabra/article/9/1/82194/197046/Ageism-in-Hiring-A-Systematic-Review-and-Meta]. Although there is more research towards ageism skewing for the older generation, we would also like to know if there are instances where the younger demographic may be feeling its adverse effects as well.

Why is it important to find this kind of bias in ML models? As hiring becomes more automated and utilizes AI/ML to sort through applicants, AI Models could potentially screen out applicants if they use language associated with older people by giving phrases with older language patterns a negative score.

For example, if a resume contained the words "Raised sales averages by 110% through rizzing up an average of 150 customers per day", the resume holder is likely to be a member of Gen Z and not Gen X.

On the other hand, if a resume contains the fact that the jobseeker graduated college in the 1970's, the AI model could flag this to the detriment of the jobseeker, meaning that this jobseeker would be filtered out because the model judged that they were not a good candidate based on using language more likely to be used by older jobseekers.

The type of investigation is relevant to other researchers because there is a growing field of interest in the intersection of social justice and AI/ML. Because the data that society generates can reflect social biases, AI/ML models that are trained on this data will enforce these biases as well. Therefore, researchers in Computer Science, Sociology, and even Law could find this sort of research to be relevant to what they are studying.

def load_embeddings(filename):
    """
    Load a DataFrame from the generalized text format used by word2vec, GloVe,
    fastText, and ConceptNet Numberbatch. The main point where they differ is
    whether there is an initial line with the dimensions of the matrix.
    """
    labels = []
    rows = []
    with open(filename, encoding='utf-8') as infile:
        for i, line in enumerate(infile):
            items = line.rstrip().split(' ')
            if len(items) == 2:
                # This is a header row giving the shape of the matrix
                continue
            labels.append(items[0])
            values = np.array([float(x) for x in items[1:]], 'f')
            rows.append(values)
    
    arr = np.vstack(rows)
    return pd.DataFrame(arr, index=labels, dtype='f')

def vecs_to_sentiment(vecs):
    # predict_log_proba gives the log probability for each class
    predictions = model.predict_log_proba(vecs)

    # To see an overall positive vs. negative classification in one number,
    # we take the log probability of positive sentiment minus the log
    # probability of negative sentiment.
    # this is a logarithm of the max margin for the classifier, 
    # similar to odds ratio (but not exact) log(p_1/p_0) = log(p_1)-log(p_0)
    return predictions[:, 1] - predictions[:, 0]

Step 4: Get a sentiment score for text¶

There are many ways to combine sentiments for word vectors into an overall sentiment score. Again, because we're following the path of least resistance, we're just going to average them.

import re
TOKEN_RE = re.compile(r"\w.*?\b")
# The regex above finds tokens that start with a word-like character (\w), and continues
# matching characters (.+?) until the next word break (\b). It's a relatively simple
# expression that manages to extract something very much like words from text.


def text_to_sentiment(text):
    # tokenize the input phrase
    tokens = [token.casefold() for token in TOKEN_RE.findall(text)]
    # send each token separately into the embedding, then the classifier
    sentiments = words_to_sentiment(tokens)
    return sentiments['sentiment'].mean() # return the mean for the classifier

def words_to_swn_sentiment_separate(pos_terms, neg_terms):
    def calculate_sentiments(terms, group_label):
        sentiments = []
        for word in terms:
            word_lower = word.lower()  # Ensure lowercase for consistency
            synsets = list(swn.senti_synsets(word_lower))
            if not synsets:
                sentiment = 0  # Neutral if word not found
            else:
                # Aggregate scores across all senses
                pos_score = sum([synset.pos_score() for synset in synsets]) / len(synsets)
                neg_score = sum([synset.neg_score() for synset in synsets]) / len(synsets)
                sentiment = pos_score - neg_score  # Simple aggregation strategy
            sentiments.append({'word': word_lower, 'sentiment': sentiment, 'group': group_label})
        return pd.DataFrame(sentiments)

    # Process positive terms
    pos_frames = []
    for group, name_list in pos_terms.items():
        pos_frame = calculate_sentiments(name_list, group)
        pos_frames.append(pos_frame)
    pos_sentiments_swn = pd.concat(pos_frames, ignore_index=True)

    # Process negative terms
    neg_frames = []
    for group, name_list in neg_terms.items():
        neg_frame = calculate_sentiments(name_list, group)
        neg_frames.append(neg_frame)
    neg_sentiments_swn = pd.concat(neg_frames, ignore_index=True)

    return pos_sentiments_swn, neg_sentiments_swn

# Calculate sentiment scores using SentiWordNet for positive and negative terms separately
pos_sentiments_swn, neg_sentiments_swn = words_to_swn_sentiment_separate(pos_terms, neg_terms)

T-statistic: The negative value suggests that, on average, the sentiment scores from the original lexicon are slightly lower than those from SentiWordNet for the common words. However, the magnitude of this difference (as indicated by the absolute value of the T-statistic) is not very large.

P-value: The P-value of 0.215 is above the common significance levels (0.05), indicating that the observed difference in average sentiment scores between the two lexicons is not statistically significant. In other words, based on this test, there is not enough evidence to reject the null hypothesis that there is no difference in the average sentiment scores provided by the two lexicons for the same set of words.

The statistical analysis suggests that, for the words common to both lexicons, there isn't a significant difference in how sentiment is scored between the original lexicon and SentiWordNet. This could mean that both lexicons are relatively consistent with each other in terms of the sentiment scores they assign to the same words, at least for the sample of words tested.

This does not discount the fact that either of these methods have resulted in wildly negative scores for both positive and negative terms used by both ends of the age gaps tested.

This is not a complete test by any means, for further testing, we would like to use another lexicon that has some of the words that we are testing for, so that the test sample is not reduced. But what it does show, is that AI teams that are producing software based on similar lexicons and word embeddings will show some bias, against the most recent types of words used as well older ones. Hence, to eliminate this bias, one has to be extremely careful when using an AI system so that it does not misinterpret you.
